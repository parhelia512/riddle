# Riddle 逃逸分析与 GC 设计

通常来说，写 Rust 时我们会被生命周期以及相关的许多问题而卡住，这会使我们大呼小叫，编译器也会大呼小叫。那我们就模仿 Go 去做一个引用逃逸机制，在引用逃逸时自动将变量提升到 GC 上来规避生命周期的问题，同时依然保有所有权机制来防止拷贝的性能消耗问题。

以下是一段示例代码，它在 Rust 中会爆出借用错误而在 Riddle 中则可以顺利执行

```rust
struct Foo{
    x: i32,
}

fn main(){
    let p: &Foo;
    {
        let s = Foo{ x: 42 };
        p = &s;
    }
    println!("{}", p.x);
}
}
```

在 Riddle 中，编译器会分析出 `p = &s;` 使得 `s` 的引用逃逸出了其所在的作用域。此时，Riddle 编译器会自动将 `s` 从栈分配提升为 GC 堆分配，从而延长其生命周期，确保 `p` 在后续使用时仍然有效。

## 逃逸分析的核心规则

Riddle 的逃逸分析遵循以下核心原则：

1. **局部性原则**：变量默认在栈上分配，所有权规则不变。
2. **逃逸检测**：当编译器检测到某个变量的引用被传递到其声明的词法作用域之外时，触发逃逸标记。
3. **自动提升**：逃逸的变量自动从栈迁移到 GC 管理的内存区域，其生命周期由 GC 控制。
4. **零开销回退**：若逃逸分析判定引用未逃逸，则依然享受栈分配的高性能和确定性的析构。
5. **跨函数逃逸**：引用作为返回值或通过闭包捕获时，同样触发逃逸分析，对涉及的所有变量进行 GC 提升。
